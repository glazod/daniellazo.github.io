Solution Architecture

Development

Data was pulled directly from ServiceNow using API endpoints across several tables: change, change_task, attachment, and sysapproval_approver. The scope was narrowed to closed change requests only, where incremental refresh was set up to grab the latest change requests added to a population saved in an extracted .csv file.

Each request in the dashboard includes a hyperlink back to ServiceNow for quick access, while attachments are also just a click away for instant download. To make things more actionable, notifications were built in to flag requests where the end date slipped past todayâ€™s date. A number of these analytics were also tied directly to internal controls, helping teams keep a closer eye on compliance and risk.
